## Notes on authored-from-synthesis

Sections built from composition synthesis on top of vendor + methodology references:

- **Three opening convictions in soul.md** — synthesize three load-bearing rules from three different sources (Outreach/Salesloft outbound-as-system, MEDDIC Academy pick-ONE-framework, Gong pipeline-hygiene-daily) into a memorable triad.
- **Sales technology categories** — names specific CRMs / enrichment vendors / sequence tools / call intelligence vendors / proposal tools / forecasting vendors from industry sources. Listing common tools is industry knowledge, not invented.
- **MEDDIC scoring rubric (0-3)** — MEDDIC Academy provides the framework; the 0-3 numeric rubric and roll-up thresholds are composition synthesis from common industry practice.
- **Cadence pattern (12-16 days, 7-9 touches)** — Outreach + Salesloft + lemlist + Instantly all converge on this; specific day-by-day pattern is composition synthesis from their reference templates.
- **Domain warmup schedule** — Lemwarm + Instantly + Smartlead all publish similar ramp schedules; composition synthesis from those references.
- **NBA heuristics (7 signals)** — Gong "Deal coaching" research provides the signals; the specific 7-signal composite and "pick ONE NBA" rule is composition synthesis.
- **Account research ICP-fit scoring (100-point)** — composition synthesis from firmographic + pain + tech-stack + trigger frameworks; specific weights (40/30/20/10) are operational choices.
- **Pipeline stage criteria table** — standard sales stage definitions; specific entry/exit criteria are composition synthesis.
- **First-conversation PROACTIVE.md self-init** — standard pattern from METHODOLOGY.md with sales-specific routine questions (CRM choice, pipeline review cadence, outbound channel).

No domain claims, performance benchmarks, or vendor capabilities were invented. All benchmarks come from vendor documentation or industry-standard sources cited above.

1. Re-fetch source files / vendor API docs cited in `reference/INVENTORY.md`
2. Diff against previous versions to see what changed
3. Update corresponding sections of `soul.md` and `role.md`
4. Update this `SOURCES.md` if section names or source URLs changed
5. Re-run `build.py` to regenerate `dist/sales-agent.craftbot`
