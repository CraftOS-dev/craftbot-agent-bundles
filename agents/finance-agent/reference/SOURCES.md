## Notes on authored-from-synthesis

Sections that aren't directly lifted from a single source (these are operational glue, not domain claims):

- **soul.md "Title + persona intro"** — synthesizes the three convictions from the per-agent prompt with `finance-controller` stylistic mirroring.
- **soul.md "Closing rule"** — synthesizes convictions + hand-off rules; no single canonical source.
- **role.md "Antipattern catalog"** — authored synthesis (8 BAD/GOOD pairs), each pair tied to a specific soul.md rule. Domain claims trace to the same sources as the underlying rule.
- **role.md "Brief / Output templates"** — authored from CFO deliverable conventions (Visible.vc Standard + NVCA review + Cube board deck) combined into 3 reusable templates.

## Refreshing from upstream

When SOTA tools change (e.g., new platform launch, NVCA update, Damodaran annual refresh):
1. Update the relevant skill pack(s) in `agents/finance-agent/skills/<name>/SKILL.md`.
2. Update the SOTA sources table above.
3. Update `reference/SOTA_USE_CASES.md` confidence ratings if applicable.
4. Re-run `python verify.py finance-agent` to confirm structure intact.
5. Re-build: `python build.py finance-agent` produces a fresh `.craftbot`.

Annual refresh triggers (always check Jan-Feb):
- Damodaran 2027 dataset (early January)
- NVCA model document updates (typically October)
- Bessemer / SaaS Capital benchmarks refresh
- Visible.vc Standard template revisions
- Tax law changes (track via Davis Wright Tremaine + Cooley + Wilson Sonsini startup law blogs)
