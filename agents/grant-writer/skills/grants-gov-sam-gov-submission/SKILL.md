# Grants.gov + SAM.gov submission

Grants.gov submission flow as of June 2026: SAM.gov entity registration must be ACTIVE → create Workspace per opportunity → complete SF-424 family forms → validate → Authorized Organization Representative (AOR) clicks Submit. The Get Opportunities Public API supports READ; the SUBMIT step remains UI-clickable. Agent prepares everything; user (AOR) submits.

Disclaimer: Submission of a federal grant application is the AOR's legally binding act. Agent does not submit on user's behalf without explicit AOR authorization.

## When to use

- User has a federal NOFO and needs to file via Grants.gov
- Org's SAM.gov registration is expired or about to expire
- First federal submission — needs end-to-end walk-through
- Multi-package submission (mandatory family forms + program-specific)

Do NOT use this skill for:
- Drafting proposal narrative (→ `full-grant-proposal-narrative-methods-evaluation`)
- Building the budget (→ `budget-narrative-justification`)
- SF-LLL lobbying disclosure (→ `sf-424-sf-lll-subaward`)
- Post-award reporting (→ `grant-reporting-interim-final`)

Auth / API key requirements:

- UEI (12-char alphanumeric) — auto-generated by SAM
- CAGE Code — generated by DLA via SAM
- Notarized authorized representative form for SAM.gov entity reg

## Common recipes

### Recipe 1: Verify SAM.gov entity registration status

```bash
# Check via SAM.gov Entity Mgmt API
curl "https://api.sam.gov/entity-information/v3/entities?\
api_key=$SAM_API_KEY&\
ueiSAM=<your_UEI>"

# Returns "registrationStatus": "Active" / "Submitted" / "Expired"
# Active expiration: 365 days from registration; renew annually
```

### Recipe 2: Renew SAM.gov registration

```markdown
## Renewal flow (UI; no API)
1. Log in at https://sam.gov
2. Workspace → Entity Management → My Entities
3. Find expiring entity → "Renew"
4. Update: legal business name, physical address, IRS EIN, NAICS codes, points of contact, banking info
5. Update Annual Representations & Certifications (FAR/DFARS reps)
6. E-sign by authorized representative
7. Wait 7-10 business days for processing
```

Don't wait until expiration; renew 60 days early.

### Recipe 3: Find an opportunity via Get Opportunities Public API

```bash
curl -X POST https://api.grants.gov/v1/api/search2 \
  -H "Content-Type: application/json" \
  -d '{
    "keyword": "maternal child health",
    "oppStatuses": "posted",
    "rows": 25
  }' | jq '.data.oppHits[] | {oppNumber, oppTitle, closeDate, agencyCode}'
```

### Recipe 4: Pull full opportunity package

```bash
curl -X POST https://api.grants.gov/v1/api/fetchOpportunity \
  -H "Content-Type: application/json" \
  -d '{"opportunityId": "352421"}' \
  | jq '{title:.data.title, agency:.data.agency, closeDate:.data.closeDate, attachments:.data.synopsisAttachments}'
```

Returns the NOFO + the application package XML schema + required forms list.

### Recipe 5: Create a Workspace for an opportunity

```markdown
## Workspace creation (UI; no API)
1. https://www.grants.gov → search opportunity
2. "Apply" → "Create Workspace"
3. Workspace name: "<Funder>-<Project>-<Year>"
4. Assign workspace owner + participants (writer, reviewer, AOR)
5. Workspace appears under My Workspaces
```

### Recipe 6: Add forms to Workspace

```markdown
## Required forms (typical discretionary grant)
- SF-424 (Application for Federal Assistance — general)
- SF-424A (Budget Information — Non-Construction)
- SF-424B (Assurances — Non-Construction)
- SF-LLL (only if lobbying activity)
- Project Narrative Attachment Form
- Budget Narrative Attachment Form
- Performance Site Location Form (NIH)
- HHS Checklist / Cover Letter (HHS)
- Other agency-specific forms

Find required forms in the NOFO's "Application Forms" section + the Workspace
auto-populates from the opportunity's XML schema.
```

### Recipe 7: Complete SF-424 online (preferred over PDF)

```markdown
## SF-424 key fields
| Field | Source |
|---|---|
| 1. Type of submission | Application |
| 2. Type of application | New / Continuation / Revision |
| 3. Date received | Auto-filled at submit |
| 5. Applicant info | From SAM (UEI, CAGE, name, address) |
| 8. Type of applicant | Nonprofit (M) — most common |
| 10. CFDA / Catalog Number | From NOFO (e.g., 93.110) |
| 12. Areas affected | County / state / country list |
| 13. Project title | Match Workspace + budget narrative |
| 14. Congressional districts | Applicant + project location |
| 15. Estimated funding | Federal + non-federal sum from SF-424A |
| 18. Authorized Rep | The AOR who will sign + submit |
| 21. SF-LLL? | Yes / No |
```

### Recipe 8: Validate Workspace before submission

```markdown
## Validation
1. Workspace → "Check Application" → fix all errors
2. Errors: missing required fields, schema validation, signature missing
3. Re-validate after every fix
4. "Application Complete" status = ready to submit
```

### Recipe 9: Submission — AOR clicks Submit

```markdown
## Submission (legally binding; AOR only)
1. AOR logs in
2. Workspace → "Sign and Submit"
3. AOR e-signs
4. Submit → tracking number returned (format: GRANT12345678)
5. Confirmation email arrives within 1 hour with full receipt
```

Agent does NOT submit. Brief AOR with checklist + watch them click.

### Recipe 11: Track submission post-submit

```bash
# Pull submission status
curl "https://api.grants.gov/v1/api/trackingNumber" \
  -X POST \
  -d '{"trackingNumber":"GRANT12345678"}'

# Returns: receipt status, agency receipt, validation status
# Log tracking number in Notion pipeline DB
```

### Recipe 12: Common Grants.gov errors + fixes

```markdown
| Error | Cause | Fix |
|---|---|---|
| Entity not registered or expired | SAM.gov lapsed | Renew sam.gov (7-10 days) |
| Invalid UEI | UEI mismatch SAM ↔ application | Confirm UEI matches SAM entity record |
| Form schema validation error | Field type mismatch | Re-validate in Workspace |
| File too large | Attachment >100MB | Compress PDF (Acrobat Optimize) |
| Special character not allowed | Em-dash, smart quotes in PDF | Re-save as plain text or print-to-PDF |
| AOR not authorized | Wrong user clicked Submit | Assign AOR role in Workspace |
| Tracking number not received | Submission failed silently | Check Workspace status; re-validate; resubmit |
```

## Examples

### Example 1: First federal submission — SAMHSA grant

**Goal:** Submit a $400K SAMHSA discretionary grant.

**Steps:**
1. Verify SAM.gov registration ACTIVE (Recipe 1). If expired, start renewal first.
2. Pull opportunity package via fetchOpportunity API; confirm CFDA + deadline.
3. Confirm AOR is assigned (org's ED or COO typically).
4. Create Workspace; assign roles (writer, reviewer, AOR).
5. Add required forms: SF-424, SF-424A, SF-424B, Project Narrative, Budget Narrative, Performance Site Location.
6. Complete SF-424 with org info from SAM entity record (Recipe 7).
7. Upload Project Narrative (final PDF + plain-text fallback).
8. Validate (Recipe 8); fix all errors.
9. 24 hours before deadline: brief AOR; walk them through Sign + Submit.
10. Track number recorded in Notion pipeline (Recipe 11).

**Result:** Submitted with tracking number; submission confirmation email archived.

### Example 2: SAM.gov renewal coordination

**Goal:** SAM.gov expires Oct 1; major federal deadline Oct 15.

**Steps:**
1. Email Operations: "Renew SAM by Sep 1, 60-day buffer".
2. Coordinate notarized authorized rep form re-sign.
3. Update FAR/DFARS reps + financial info if changed.
4. Monitor status daily until Active.
5. Verify Active via Recipe 1.
6. Proceed with Workspace creation only after Active.

**Result:** SAM Active by Sep 10; safe buffer for Oct 15 deadline.

## Edge cases / gotchas

- **SAM.gov renewal takes 7-10 business days; sometimes 3+ weeks if mismatch.** Plan 60 days out.
- **AOR vs Workspace owner.** Workspace owner can edit; AOR can submit. Often the same person but not always.
- **UEI replaced DUNS in 2022.** Old NOFOs may say DUNS — UEI is the live identifier.
- **Notarized form required for SAM.** New entity registration requires notarized authorized representative form. Allow time.
- **System for Award Management (SAM) vs SAM.gov.** Same thing; "System for Award Management" is the legacy name.
- **Federal holidays delay processing.** SAM.gov processing pauses on federal holidays. Plan around.
- **Re-submission allowed before deadline.** Submit early → if errors, fix and re-submit. Latest submission counts.
- **NOFO updates / amendments.** Agencies amend NOFOs; subscribe to opportunity to receive amendment emails.
- **Required attachments format:** PDF preferred; some forms require Word; check each form's spec.
- **NIH eRA Commons separate from Grants.gov.** NIH applications transmit through eRA Commons after Grants.gov receipt. Different login.
- **HHS / NIH page count and formatting:** strict — 11pt Arial/Helvetica, 0.5" margins, single-spaced. Some NOFOs cap font + margins. Read the NOFO instructions.
- **Disclaimer:** AOR signature is legally binding. Agent never submits.
