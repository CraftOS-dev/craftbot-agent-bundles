# Customer Interview Script + Synthesis — SKILL

End-to-end interview pipeline: script → schedule → record → transcribe → tag → synthesize. Lenny Rachitsky's 5-question discovery template, the JTBD timeline, the Mom Test guardrails, and Dovetail tagging form the spine.

## When to use

- Designing a customer interview script (5 questions or full JTBD timeline).
- Recruiting and scheduling 5-15 interviews on a single JTBD.
- Running the interview itself (live in Zoom; recorded via Fathom/tl;dv/Otter).
- Synthesizing transcripts into themes (handoff to `dovetail-research-synthesis`).
- Building the research repo entry that PRDs cite.

Trigger phrases: "schedule customer interviews", "write an interview guide", "JTBD interview", "synthesize these calls", "discovery script", "I want to talk to 5 customers about X".


## Common recipes

### Recipe 1: 5-question discovery script (Lenny template)

```markdown
# Interview Guide — [Topic / JTBD]

**Researcher:** [name] · **Goal:** [what we're trying to learn]
**Sample target:** 5-8 interviewees, [segment]
**Duration:** 30 minutes
**Recording consent:** "May I record? Recording is used internally only."

## Pre-interview
- Confirm role, team size, current workflow on [topic] in 60 seconds.

## The 5 questions

1. **Walk me through the last time you [did the job].**
   - When was it? What triggered it? What was the situation?
   - (Goal: surface the actual moment + push forces)

2. **What did you try first? What other options did you consider?**
   - What did you reject? Why?
   - (Goal: alternative set + non-consumption competitors)

3. **What was easy? What was hard? What almost stopped you?**
   - (Goal: pull + anxieties + habits)

4. **What did you hope would change? What actually changed?**
   - (Goal: outcome + delta to expectation)

5. **If you could wave a magic wand, what would the perfect version of [this job] look like?**
   - (Goal: unconstrained desired outcome — generates JTBD outcome statements)

## Probes (use anytime)
- "Tell me more about that."
- "Can you give me a specific example?"
- "What happened next?"
- "Why is that important to you?"

## Mom Test guardrails
- ❌ Don't pitch your idea.
- ❌ Don't ask "would you use X" (compliments lie).
- ✅ Ask about the past, not the future.
- ✅ Ask "what" and "how", avoid "why" (sometimes).
- ✅ Get to specific stories with dates and details.
```

### Recipe 2: JTBD timeline interview (full version)

```markdown
# JTBD Timeline Interview — [Job]

## The forces of progress to surface
- Push: current situation pushes them
- Pull: new solution pulls them
- Anxiety: fear of switching
- Habit: comfort with current

## Timeline questions
1. **First thought** — "When did you first think you needed a new way to [do job]?"
2. **Passive looking** — "What were you doing about it before you actively searched?"
3. **Active looking** — "What did you do to actively find a solution?"
4. **Deciding** — "How did you narrow it down? What turned you off other options?"
5. **First use** — "Walk me through the first time you used [solution]."
6. **Ongoing use** — "What does using it look like now? What worked? What didn't?"
7. **Decision retrospective** — "Knowing what you know now, would you make the same choice?"
```

### Recipe 9: Recruitment via PostHog cohort + Gmail blast

```bash
# Pull a high-intent cohort from PostHog
COHORT_USERS=$(mcp tool posthog.query --query "
  SELECT email FROM persons
  WHERE properties.plan='pro'
    AND properties.last_active >= now() - INTERVAL 30 DAY
  LIMIT 50
" | jq -r '.results[].email')

# Send outreach via Gmail
echo "$COHORT_USERS" | while read email; do
  mcp tool gmail.send --to "$email" \
    --subject "Quick chat about your workflow?" \
    --body "$(cat outreach-template.md)"
  sleep 5  # gentle pacing
done
```

## Examples

### Example 1: Schedule 8 interviews this week
**Goal:** Get 8 booked interviews on a single JTBD.

**Steps:**
1. Generate the script (Recipe 1) into Notion (Recipe 3).
2. Create the Calendly event (Recipe 4).
3. Pull recruitment cohort from PostHog (Recipe 9).
4. Send outreach (Recipe 5).
5. Track bookings; aim for 8 by end of week.
6. Send confirmation + reminder email day-of via `gmail-mcp`.

**Result:** 8 booked customer interviews with a script and a recording plan.

### Example 2: From transcript to PRD
**Goal:** Turn 8 transcripts into a PRD problem section.

**Steps:**
1. Auto-upload Fathom VTTs to Dovetail (Recipe 6 in a loop).
2. Run Dovetail auto-tag + manual cleanup (see `dovetail-research-synthesis` Recipe 3).
3. Build 3-7 themes; count occurrences (Recipe 5 of Dovetail skill).
4. Create the research repo entry (Recipe 8 here).
5. Generate the PRD draft (Recipe 10) that cites the research.

**Result:** PRD with cited, defensible problem statement.

## Edge cases / gotchas

- **Incentives.** Standard rate: $50 Amazon gift card for 30 min. Higher for senior buyers ($100-200). Without incentive, response rates drop to 5-10%.
- **Self-report ≠ behavior.** Always ask about specific past actions, not hypothetical "would you" or "do you think". This is the Mom Test core.
- **5+ interviews threshold.** Don't synthesize themes from <5 interviews; you'll over-fit to a single voice.
- **Recording consent.** Always ask at the start. GDPR/CCPA: store transcripts with consent record.
- **Calendly auto-cancellation.** No-shows happen at 15-25%. Always recruit 1.3x your sample target.
- **Outreach blast → spam folder.** Pace at ~5 emails/min from Gmail; or use `klaviyo-email-lifecycle` if recruiting from a large list.
- **Fathom/Otter/tl;dv accuracy.** ~85-95% on clean audio; mumbled or noisy interviews degrade. Spot-check transcripts.
- **Pitching during interviews.** PMs slip into pitching ~30% of the way through. Re-read the Mom Test guardrails before each call.
- **Single-segment bias.** Don't interview only your power users — recruit churned users, free-tier users, and prospects.
- **Don't generate the script and then never edit it.** Always tailor the 5 questions to your hypothesis. The template is a frame, not a script.
