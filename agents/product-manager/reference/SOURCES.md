## Notes on "authored from synthesis"

Several sections include composition synthesis on top of the referenced material:

- **Three opening convictions in soul.md** — synthesized from three load-bearing rules: Cagan "outcomes over outputs", Cagan/Torres "roadmap as hypothesis", JTBD canon "research is data, not opinions". Triad framing is composed.
- **PRD 1-pager and full templates in role.md** — synthesized from Lenny + Reforge + ChatPRD elements (problem / hypothesis / primary user / success / scope / non-goals / open Qs / risks / GTM / tracking). The specific section order is composed; each section's existence is from sources.
- **PRD review rubric** — synthesized from ChatPRD + Kraftful + Reforge rubric items. The 11-item checklist is composed; each item maps to a source.
- **Roadmap one-pager template** — synthesized from ProductPlan + Productboard + Lenny templates. Section order is composed.
- **Stakeholder update template (Wins / Lowlights / Asks / Plans / Metrics)** — from Lenny's exact template.
- **Launch checklist (P0/P1/P2 + tiered subsections)** — synthesized from Reforge + SVPG checklists. P0/P1/P2 tiering is composed; subsections from sources.
- **Antipattern catalog** — composed from Cagan SVPG + Lenny + Reforge + Kohavi anti-patterns commentary; specific BAD/GOOD pairs are illustrative composition.
- **First-conversation PROACTIVE.md self-init** — standard pattern from METHODOLOGY.md with PM-specific routine questions.

No domain claims, performance benchmarks, framework formulas, or methodology canon were invented. RICE/ICE/Kano/MoSCoW/WSJF formulas, JTBD outcome statement structure, Rumelt kernel, 7 Powers, dual-track agile, Patton story mapping — all are cited canon.

## How to update this agent

1. Re-pull SOTA tool docs (Linear / Notion / Figma / Dovetail / Maze / Amplitude / Mixpanel / PostHog / Statsig / GrowthBook / Lattice / FullStory / LogRocket / Centercode / Productboard) every quarter — SOTA changes monthly.
2. Diff against previous versions; update `reference/SOTA_USE_CASES.md` confidence ratings.
3. Update corresponding sections of `soul.md` and `role.md`.
4. Update this `SOURCES.md` if section names or source URLs changed.
5. Re-run `python verify.py product-manager` to confirm structure intact.
6. Re-run `python build.py product-manager` to regenerate `dist/product-manager.craftbot`.

For canonical PM reference repos:
- `wshobson/agents` — repull every quarter for any new PM agent definitions.
- `VoltAgent/awesome-claude-code-subagents` — same cadence.
- `msitarzewski/agency-agents` — same cadence.
