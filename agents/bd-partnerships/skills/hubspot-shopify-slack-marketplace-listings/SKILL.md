# HubSpot / Shopify / Slack / OpenAI / Stripe / Atlassian / AppSource Marketplace Listings — SKILL

Publish (and maintain) listings across the SaaS-side marketplaces: HubSpot App Marketplace, Shopify App Store, Slack App Directory, OpenAI GPT Store, Stripe Marketplace, Atlassian Marketplace (Cloud + Data Center), Microsoft AppSource. Pattern is similar across all: developer-program enrollment → app/integration build → listing-asset bundle → portal upload → manual review → publish.

## When to use

- **Launching on any non-cloud SaaS marketplace** (excludes AWS/Azure/GCP/AppExchange — see other skills).
- **Updating listing assets** — screenshots, description, pricing.
- **Maintaining OAuth scopes / manifest config** — review every quarter.
- **Adding new app type to existing marketplace** (e.g., HubSpot adds Workflow Action, Slack adds Slash Command).
- **Cross-marketplace launch coordination** — same product, 5 listings, consistent messaging.
- **Trigger phrases**: "HubSpot app listing", "Shopify app store", "Slack App Directory", "GPT Store", "Stripe Marketplace app", "Atlassian Marketplace", "AppSource SaaS offer".

Do NOT use this skill for: **cloud marketplaces** (use `aws-azure-gcp-marketplace-listings`); **Salesforce** (use `salesforce-appexchange-listing`); **app architecture** (defer to `tech-lead-cto`).

## Setup

```bash
# Managed OAuth via Maton
export MATON_API_KEY="<key>"

# Direct API keys (one per marketplace)
export HUBSPOT_DEV_API_KEY="<key>"          # HubSpot Developer Account → App settings
export SHOPIFY_PARTNERS_API_KEY="<key>"     # Shopify Partners Dashboard → API
export SLACK_APP_TOKEN="xapp-..."           # Slack App Config Token
export OPENAI_API_KEY="<key>"               # for testing GPT Actions
export STRIPE_SECRET_KEY="sk_live_..."      # Stripe Apps SDK or Connect
export ATLASSIAN_OAUTH_TOKEN="<token>"      # Atlassian Marketplace
export MS_PARTNER_CENTER_TOKEN="<token>"    # Partner Center for AppSource

# Playwright for portal-only flows
# (most marketplaces have at least one portal-only step)
```

**One-time human tasks:**
- HubSpot: Developer Account at https://developers.hubspot.com.
- Shopify: Shopify Partners account at https://partners.shopify.com.
- Slack: api.slack.com → Create New App.
- OpenAI: ChatGPT Plus or Enterprise; Builder Profile.
- Stripe: Stripe account + Apps SDK access.
- Atlassian: Marketplace Vendor account.
- Microsoft AppSource: Partner Center enrollment + MPN ID.

## Common recipes

### Recipe 1: HubSpot App — register + OAuth config

```bash
# Apps are created in dev portal first; then OAuth credentials returned
# Manifest tweaks via Developer API:
curl -X POST "https://api.hubapi.com/integrations/v1/<dev-id>/apps" \
  -H "Authorization: Bearer $HUBSPOT_DEV_API_KEY" -H "Content-Type: application/json" \
  -d '{
    "name":"Brand Analytics",
    "description":"Real-time analytics for HubSpot revenue teams",
    "redirectUrls":["https://brand.com/hubspot/oauth/callback"],
    "scopes":["crm.objects.contacts.read","crm.objects.deals.read","timeline","oauth"]
  }'
```

After registration: developer portal → App listing → fill marketplace fields → submit. Reference: https://developers.hubspot.com/docs/api/creating-an-app.

### Recipe 11: Stripe Connect — payouts to partners (Stripe Marketplace pattern)

```bash
# Stripe Connect = payout platform for marketplaces (e.g., your app processes payments on behalf of partners)
curl -X POST "https://api.stripe.com/v1/accounts" \
  -H "Authorization: Bearer $STRIPE_SECRET_KEY" \
  -d "type=express" -d "country=US" -d "email=partner@acme.com"

# Then create transfers as payouts:
curl -X POST "https://api.stripe.com/v1/transfers" \
  -H "Authorization: Bearer $STRIPE_SECRET_KEY" \
  -d "amount=15000" -d "currency=usd" -d "destination=acct_xxx" \
  -d "description=Commission for Acme — June 2026"
```

Reference: https://docs.stripe.com/connect.

## Examples

### Example 1: Cross-marketplace launch of new SaaS

**Goal:** New SaaS analytics product launches simultaneously on HubSpot, Slack, OpenAI GPT Store.

**Steps:**
1. Day 0 — Build universal asset registry (Recipe 14).
2. Day 1 — Recipe 1 + 2 + 3 → HubSpot listing submitted.
3. Day 1 — Recipe 7 + 8 → Slack listing submitted.
4. Day 1 — Recipe 9 → OpenAI GPT submitted (publishes faster — often days).
5. Day 7 — GPT live; OpenAI traffic generates first installs.
6. Day 21 — HubSpot + Slack live.
7. Day 22+ — Cross-marketplace install tracking via Recipe 15.

**Result:** Coordinated launch across 3 marketplaces; first traffic in <1 week via GPT Store.

### Example 2: Built for Shopify badge pursuit

**Goal:** Shopify app live with 80 installs + 4.6 rating; want BFS badge.

**Steps:**
1. Recipe 6 — Review BFS checklist; identify gaps.
2. Engineering work: embedded admin UI (was external redirect); webhooks (was polling).
3. Lighthouse audit; performance score 78 (passes ≥70).
4. Accessibility audit + fixes.
5. Apply via Shopify Partners portal.
6. ~6 week review; BFS badge granted.

**Result:** BFS badge unlocks featured placement; install rate 3x post-badge.

### Example 3: AppSource Co-Sell Ready for enterprise

**Goal:** Already on AppSource as transactable SaaS; pursue Co-Sell Ready.

**Steps:**
1. Customer reference list: 3 enterprise Azure customers using product.
2. Reference architecture diagram via `drawio-mcp`.
3. Solution overview deck via `pptx`/`canva-mcp`.
4. Submit via Partner Center → Co-Sell tab.
5. 6-12 week review.
6. Granted → Microsoft sellers pitch joint deals.

**Result:** Co-Sell Ready unlocks Microsoft field-sales motion; new pipeline 2-3x organic.

## Edge cases / gotchas

- **Marketplace review timelines vary**: OpenAI GPT Store — days; HubSpot — 2-4 weeks; Slack — 2-4 weeks; Shopify — 1-3 weeks; AppSource Co-Sell — 6-12 weeks.
- **Asset rejections** common: image size off-spec, screenshot containing sensitive data, demo video too long, missing accessibility text. Pre-check rigorously.
- **HubSpot scopes are immutable post-publish** — adding scopes triggers re-review. Plan OAuth scope carefully at launch.
- **Shopify "Built for Shopify"** requires Polaris UI + embedded experience — major engineering investment, not just a flag.
- **Slack manifest changes don't always require resubmission** — only "user-facing" changes (description, icon, scopes) do.
- **OpenAI GPT Store domain verification** requires DNS TXT record + 24-48h propagation.
- **OpenAI GPT actions can't access user credentials directly** — must use OAuth or API key headers.
- **Stripe Apps run inside Stripe Dashboard iframe** — auth + UX patterns are unique.
- **Stripe Connect "Express" vs "Custom" account types** — Express has minimal compliance burden; Custom gives full white-label but you own KYC.
- **Atlassian Marketplace has Cloud + Data Center + Server** — Cloud only is fine for modern; Data Center if customers are enterprise self-hosted.
- **AppSource is Partner-Center-shared with Azure Marketplace** — same backend, different listing surface.
- **MPN ID is required** for Microsoft listings — obtain via Microsoft Partner Network enrollment (free tier exists).
- **All marketplaces have privacy + terms URL requirements** — must be publicly accessible at submission. Broken URLs auto-reject.
- **Pricing localization**: most marketplaces support multi-currency; some auto-convert from USD; some require explicit localization.
- **Refund + cancellation** policies often required — write a clear policy URL before submission.
- **App impersonation / OAuth security**: most reviewers test OAuth flow + token scopes — pre-test that your OAuth correctly validates state parameter, doesn't leak tokens in URLs, etc.
- **Listing analytics quality varies** — HubSpot has best dashboards, Slack offers basic, GPT Store has none (rely on Action API logs).
- **Free tier vs paid tier conversion**: explicit free → paid prompt in onboarding lifts conversion 30-50%; build it.
- **Cross-marketplace user-data export**: GDPR-mandated; most marketplaces require explicit data-deletion endpoint. Build once, reuse.
- **Co-marketing budget**: most marketplaces offer co-marketing fund (Slack Partnership Fund, HubSpot ecosystem fund) — apply once published; ~$5-25K available.
