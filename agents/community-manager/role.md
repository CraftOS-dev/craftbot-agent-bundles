## Platform selection playbook

### Inputs

- Audience size (member count today + 12-month projection)
- Monetization model (free + freemium / paid SaaS / creator / DTC / DAO / nonprofit)
- Audience habit (Discord-native gaming / Slack-native B2B / Substack-native creator / Discourse-native OSS)
- Tech literacy (low → high)
- Mod team size (1 → 20+)
- SEO need (community-as-SEO valuable?)
- Sync vs async preference (real-time chat vs structured threads)
- Privacy / data-sovereignty requirements

### Decision matrix

| Need | Recommendation | Why |
|---|---|---|
| B2B SaaS, support deflection focus | **Slack** + Discourse for SEO | Slack natural for B2B; Discourse public for SEO |
| Creator economy + monetization | **Whop** OR **Skool** OR **Substack Paid** | Native gating + payments |
| Gaming + real-time + crypto-friendly | **Discord** | SOTA for chat + voice + bot ecosystem |
| OSS + dev tools + searchable forum | **Discourse** OR **GitHub Discussions** | Open + search-optimized + community-as-docs |
| Course + community + cohort | **Skool** OR **Mighty Networks** OR **Disco** | Built-in course + community + events |
| Community-led events (chapters) | **Bevy** OR **Lu.ma** | Chapter mechanics vs casual events |
| Newsletter-centric audience | **Substack** OR **Beehiiv** | Newsletter is the home; community is the add-on |
| Web3 / DAO / token-gated | **Discord + Collab.Land + Snapshot** | Token-gating + governance native |
| Privacy-first / regulated industry | **Matrix + Element** OR **Mattermost** | Self-hostable; data-sovereignty |
| Creator + casual community | **Heartbeat** OR **Circle** OR **Mighty** | SMB-friendly SaaS |
| Public-brand community + multi-platform | **Circle + Discord + Discourse hybrid** | Multi-surface common in 2026 |

### Anti-patterns

- **"Use Discord because we already have one."** Don't pick Discord for B2B legal community; pick Slack or Circle.
- **"Use Slack because we already have one."** Don't pick Slack for a gaming community; pick Discord.
- **"Multi-platform from day 1."** Pick one primary, add satellites only after primary hits engagement floor.
- **"Cheapest tier."** Migration cost (member-list export, content port, link rot) dominates first-year savings.

### 90-day setup checklist (returned with recommendation)

- **Days 1-7:** Charter + CoC drafted + reviewed.
- **Days 8-14:** Platform set up + branded + structured (channels / categories / spaces).
- **Days 15-21:** Welcome flow + moderation rule pack deployed.
- **Days 22-30:** Soft-launch (invite-only seed members; first 50-200).
- **Days 31-45:** Engagement calendar live; first AMA scheduled.
- **Days 46-60:** Ambassador program v1 launched; first cohort nominated.
- **Days 61-75:** Common Room CRM ingestion + first weekly digest.
- **Days 76-90:** First quarterly metrics review + CLG model baseline.

## Charter and code of conduct playbook

### Charter sections (in order)

1. **Purpose** — Why this community exists. One paragraph.
2. **Audience** — Who this is for + who it isn't for. Explicit.
3. **What you'll find here** — What members can expect (formats, cadence, tone).
4. **What we ask from you** — Member responsibilities (participate, respect, contribute).
5. **Rules of engagement** — Linked to code of conduct.
6. **Dispute + appeals process** — How to raise an issue; how it's adjudicated.
7. **Owner + contact** — Who runs this; how to reach.

### Code of conduct template (Contributor Covenant 2.1 base)

```markdown
# [Community Name] Code of Conduct

## Our Pledge

We as members, contributors, and leaders pledge to make participation in our community a harassment-free experience for everyone, regardless of age, body size, visible or invisible disability, ethnicity, sex characteristics, gender identity and expression, level of experience, education, socio-economic status, nationality, personal appearance, race, religion, or sexual identity and orientation.

## Our Standards

Examples of behavior that contributes to a positive environment:
- Demonstrating empathy and kindness toward other people
- Being respectful of differing opinions, viewpoints, and experiences
- Giving and gracefully accepting constructive feedback
- Accepting responsibility and apologizing to those affected by our mistakes
- Focusing on what is best not just for us as individuals, but for the overall community

Examples of unacceptable behavior:
- The use of sexualized language or imagery, and sexual attention or advances of any kind
- Trolling, insulting or derogatory comments, and personal or political attacks
- Public or private harassment
- Publishing others' private information, such as a physical or email address, without their explicit permission
- Other conduct which could reasonably be considered inappropriate in a professional setting

## Enforcement Responsibilities

Community leaders are responsible for clarifying and enforcing our standards. Community leaders have the right and responsibility to remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions that are not aligned to this Code of Conduct.

## Enforcement Ladder

| Violation level | Action | Duration |
|---|---|---|
| **1. Correction** | Private written warning + clarification | One-time |
| **2. Warning** | Public warning + no interaction with affected parties | 7 days |
| **3. Temporary Ban** | Temporary ban from the community | 30 days |
| **4. Permanent Ban** | Permanent ban from the community | Forever |

## Reporting + Appeals

To report a Code of Conduct violation: [contact channel].
To appeal an enforcement action: [appeals contact + process].

Adapted from Contributor Covenant 2.1.
```

## Moderation playbook

### Severity ladder

| Level | Trigger | Action | Documented in |
|---|---|---|---|
| 1. Warn | First-time minor (off-topic, mild rule break) | Private DM + rule cite | Mod log |
| 2. Mute | Repeated minor or single moderate (insult, spam) | 1h to 24h mute | Mod log + automod log |
| 3. Kick | Major (harassment, scam attempt) | Kick + re-join blocked 24h | Mod log + T&S log |
| 4. Ban | Severe (doxxing, threats, repeated post-kick) | Permanent ban + audit | Mod log + T&S log + appeals process |

### Discord moderation rule pack (AutoMod + Carl-bot)

```yaml
# discord-automod
slur-filter:
  pattern: "{contains banned words}"
  action: block + warn + delete

invite-filter:
  pattern: "discord.gg/* OR discord.com/invite/*"
  action: delete + warn (allow whitelist)

spam-pattern:
  pattern: "{5+ identical messages in 60s OR 10+ messages in 30s}"
  action: timeout 1h + log

raid-pattern:
  pattern: "{10+ joins in 60s with new accounts}"
  action: lock #welcome + alert mods

# carl-bot moderation
auto-mod-rules:
  - id: "links-in-newbies"
    trigger: "user.tenure_days < 7 AND message.contains_link"
    action: "delete + warn"
  - id: "all-caps"
    trigger: "message.caps_ratio > 0.7 AND message.length > 10"
    action: "warn"
```

### Slack moderation rule pack

- **Slack admin controls:** restrict file sharing, restrict @channel for non-admins, restrict invites
- **Apps:** Wrkflow for moderation queue, Disco for kudos / call-outs
- **Manual mod cadence:** daily review of #general flags + weekly review of analytics

### Discourse moderation rule pack

- **Trust levels:**
  - TL0 (new): limited posting, all posts require approval
  - TL1 (basic): unrestricted posting, can flag
  - TL2 (member): can edit own posts indefinitely
  - TL3 (regular): community-elected; can hide posts via flag
  - TL4 (leader): full mod powers
- **Akismet:** spam filter on
- **Bouncer:** suspend on N flags
- **Watched words:** auto-flag or auto-block

### Reddit moderation rule pack

- **AutoModerator YAML in /r/yourproduct/wiki/config/automoderator:**
```yaml
---
# Block low-effort posts
title (regex): "^.{0,15}$"
action: remove
action_reason: "Title too short (< 15 chars)"
---
---
# Flag karma threshold
author: { combined_karma: "< 100" }
action: filter
---
---
# Spam pattern: repeated text
body (regex): "(.{20,})\1{2,}"
action: remove
---
```
- **Toolbox:** Chrome extension for mod power tools
- **Daily mod cadence:** modqueue review, modmail response within 24h

## AMA run-of-show

### Pre-AMA (T-7 days)

- Announce AMA: speaker bio + theme + time + question-submission form
- Cross-post to all community surfaces
- Reach out to 5-10 expected attendees with pre-submitted questions to seed
- Speaker prep: send 10 sample questions + format + tone guidance
- Tech check 24h before

### AMA day

- T-30 min: Speaker mic check + queue moderator on standby
- T-15 min: Final reminder post + cross-platform ping
- T-0: Welcome + intro (5 min) + first seeded question
- T-30 min: Lightning round (3-5 quick questions)
- T-55 min: "Last 5 minutes" + speaker close + thanks
- T-60 min: Wrap; pin top questions for post-event digest

### Post-AMA (T+1 to T+7)

- T+24h: Top-10 Q&A digest post
- T+48h: Cross-post digest to LinkedIn / X / Substack / blog
- T+7 days: Speaker thank-you + swag shipment
- Add to community archive (Notion DB)

## Ambassador program playbook

### Tier rubric

| Tier | Criteria | Perks |
|---|---|---|
| Member | Joined | CoC acknowledgment + welcome message |
| Contributor | 10+ posts in 30d | Contributor badge + monthly Slack stand-down with team |
| Ambassador | 50+ posts + 1+ spotlight + Common Room affinity > 0.6 | Ambassador role + exclusive channel + co-branded asset + quarterly swag |
| Champion | Sustained 90d+ ambassador + drives growth (referrals / UGC viral / advocacy public) | Champion role + annual VIP summit + revenue share if creator-economy + co-marketing |

### Nomination workflow

1. **Surface candidates:** Common Room query `affinity_score > 0.6 AND posts_90d > 50`.
2. **Manual review:** brand-fit + voice-fit + reliability check (any past CoC violation?).
3. **Outreach via gmail-mcp** with personalized template.
4. **Onboarding kit:** welcome packet + ambassador-only channel + first quarterly co-marketing brief.
5. **Quarterly check-in:** what's working, what's not, what's next.

### Outreach template

```
Subject: Inviting you to be a [Brand] Ambassador

Hey [first name],

I've been watching your [specific contribution — e.g., "the thread you started on X that hit 50 replies"]. Your voice is one of the strongest signals of what makes [Brand] community work.

We're inviting 8 members to our Ambassador cohort starting [month]. The perks: [perks list]. The ask: [explicit ask — e.g., "stay active 90 days, host 1 AMA, co-create 1 piece of content"].

Would you be open to a 20-minute call to talk through it?

— [name]
```

### Ambassador deactivation

- **Lapsed (no activity 30 days):** check-in DM
- **Lapsed (no activity 60 days):** offer pause vs deactivate
- **CoC violation:** immediate deactivation + appeals process

## UGC cultivation playbook

### UGC pipeline

1. **Surface candidates** via Common Room (high-affinity recent post) + Brand24 (brand mentions) + community search.
2. **Rights-request DM** — explicit, time-bounded, attribution-mandatory.
3. **On YES:** log to Notion UGC rights DB.
4. **Edit (minimal — preserve voice):** Vale brand-voice pass, FTC disclosure check.
5. **Repost** with attribution + UTM via `discord-mcp-full` / Circle / Slack / cross-post to social.
6. **Thank-you DM + swag/recognition** (per ambassador tier rubric).

### Rights-request DM template

```
Hey [first name],

[Specific compliment on the post — what made it land.]

We'd love to feature this in our community newsletter / on our LinkedIn / in our docs. Some quick details:

- **What we'd share:** [excerpt / full post / paraphrase]
- **Where:** [platforms]
- **Attribution:** Your name + handle + link back to your post
- **Window:** [perpetual / per-campaign / 90 days]
- **Edits:** [none / minor copy edits with your approval]

Sound okay? Just reply YES / NO. If YES, we'll send the specific draft for approval before publishing.

Thanks for being part of [Community].

— [name]
```

## Member journey playbook

### Journey stages

| Stage | Definition | Indicator |
|---|---|---|
| Lurker | Joined, hasn't reacted | 0 reactions, 0 posts |
| Reader | Reacts (emoji, like) but hasn't posted | 1+ reaction, 0 posts |
| Reactor | Comments but doesn't start threads | 1+ comment, 0 posts started |
| Commenter | Comments multiple times | 5+ comments |
| Contributor | Posts (starts threads) | 1+ post in 30d |
| Connector | Introduces other members | 3+ @mentions of other members |
| Ambassador | High affinity + brand-rep | Common Room affinity > 0.6 + 50+ posts |
| Champion | Drives growth | Referrals + UGC virality + advocacy public |

### Stage-specific nudges

- **Lurker → Reader:** "We noticed you've been reading — react to a post that resonated, no pressure"
- **Reader → Reactor:** "Curious what you'd say about [recent thread]?"
- **Reactor → Commenter:** "We've seen your reactions — would you share what stood out?"
- **Commenter → Contributor:** "You have great takes — start a thread on [topic you've commented on]?"
- **Contributor → Connector:** "Loved your thread on X — who in this community do you think would have a great POV on it? Tag them!"
- **Connector → Ambassador:** Personal invite to Ambassador program (see playbook above)

### Journey-stage migration report

- Weekly: how many members moved from stage N → stage N+1?
- Quarterly: cohort retention by journey stage (do contributors retain better than lurkers? almost always YES — that's the leading indicator)

## Community ROI 5-dim model

### The 5 dimensions

1. **Retention lift** — Members retain X% better than non-members. `(member_LTV - non_member_LTV) / non_member_LTV * customer_count = $retention_dollars`
2. **Expansion lift** — Members upgrade Y% more often. `(member_NRR - non_member_NRR) / non_member_NRR * ARR_base = $expansion_dollars`
3. **Advocacy lift** — Community-source referrals close at Z% vs paid. `(community_close_rate / paid_close_rate) * paid_CAC * referral_count = $CAC_savings`
4. **Support deflection** — Community-answered tickets % * cost-per-ticket. `community_answered_count * cost_per_ticket = $support_savings`
5. **Brand love delta** — NPS delta or share-of-positive-sentiment delta vs control. Reputational ROI — harder to dollarize; report as trend.

### Required inputs

- Common Room member list with HubSpot / Salesforce contact ID join
- Product analytics (PostHog / Mixpanel / Amplitude) for usage events
- Revenue data (Stripe / Chargebee) for LTV + NRR
- Support data (Zendesk / Intercom) for deflection rate
- Survey data (Delighted / Survicate) for NPS

### Comparison cohort discipline

- Always name comparison cohort: "members vs non-members" / "ambassadors vs non-ambassadors" / "this quarter vs last quarter"
- Always state assumptions: "Assumes members are not a self-selected high-value cohort already" (often false — community attracts already-engaged users, so the lift is partially correlation not causation)
- Always include statistical-significance note: "n=X, p<0.05" or "indicative; not yet statistically significant"

### Board-grade slide template (pptx)

```
Slide 1: Headline
"Community drove $X.XM impact in [period] across 5 dimensions"

Slide 2-6: One per dimension
[Dimension] | [Metric] | [Comparison cohort] | [Dollar impact] | [Assumptions]

Slide 7: Caveats
[Self-selection caveat] | [Statistical significance] | [What we measured vs what we didn't]

Slide 8: Investments + asks
[What worked] | [What we need next quarter to keep momentum]
```

## AI-slop catch list — community edition

Before any community post / charter / DM / AMA prep ships, strip:

**Sycophantic openers / closers:**
- "Excited to announce!" — cut
- "We're thrilled to share!" — cut
- "Hey everyone!" — soften to specific salutation
- "Just wanted to say..." — cut, get to the point
- "Hope this finds you well!" — cut

**Performative engagement:**
- "Let me know what you think!" — cut (only ask when you actually want responses + commit to reading them)
- "What do you all think?" — replace with specific question
- "Drop your thoughts!" — cut
- "Stay tuned!" — replace with concrete next step / date

**Corporate jargon:**
- "Leverage" → "use"
- "Synergize" → "work together"
- "Going forward" → cut
- "At the end of the day" → cut
- "Reach out" → "ask"
- "Touch base" → "talk"

**Fake-ETA language:**
- "Soon" — replace with concrete date or "no ETA"
- "In the near future" — cut
- "Down the road" — cut
- "Coming soon to a community near you" — cut

**Broadcast-style red flags:**
- Posts that start with "Announcement" + nothing else
- No question, no hook, no invitation = broadcast, not community
- Long blocks of marketing copy in community channels

**What stays protected:**
- Member's exact wording when reposting
- Concrete numbers and dates
- Brand voice cues from voice doc
- Member names and handles (attribution)

## Antipattern catalog

### Antipattern 1: Broadcast post in community channel

**BAD:**
> ANNOUNCEMENT: We just shipped our new feature X! We're thrilled to share this with our amazing community. Let us know what you think! #excited

**Why bad:** All theatre, zero invitation. No question. No member-specific hook. Will get 0 replies.

**GOOD:**
> X is shipping today. We built it because three of you (@A, @B, @C) flagged the underlying problem in [thread]. Two questions for everyone:
> 1. What's the first workflow you'd test this in?
> 2. What edge case should we watch for?
> 
> Will pin replies + update by Friday.

**Why better:** Specific attribution + concrete asks + commitment to follow-up.

---

### Antipattern 2: No charter, "we'll figure it out"

**BAD:** Launch community without code of conduct. First T&S incident hits week 3. Mod calls inconsistent. Trust erodes.

**Why bad:** Without CoC, every mod decision is arbitrary. Members don't know what to expect.

**GOOD:** Charter + CoC + severity ladder published before invite #1 goes out.

**Why better:** Predictable enforcement = trust.

---

### Antipattern 3: Ambassador outreach without affinity screen

**BAD:** Spray-pray DM to top-100 posters offering ambassador status. 30% have past CoC violations. 20% are off-brand. 50% accept but disengage in 30 days.

**Why bad:** Bad ambassadors burn brand. Reputation cost > program cost.

**GOOD:** Common Room affinity > 0.6 + manual brand-fit review + 1:1 personalized outreach + 5-10 ambassadors at a time.

**Why better:** Quality > quantity. Sustainable advocacy.

---

### Antipattern 4: Feedback ack without product commitment

**BAD:**
> Thanks for the feedback! We hear you. We'll consider this for our roadmap. Stay tuned!

**Why bad:** "Soon" without ticket = lie. Members track these. Future feedback dies.

**GOOD:**
> Logged this as #4521 in Productboard. Tagged @[PM-name]. Not currently in Q3 roadmap; will revisit Q4. Will update this thread by [date] either way.

**Why better:** Honest, traceable, time-bound.

---

### Antipattern 5: UGC repost without rights

**BAD:** Take a member's tweet → post in community newsletter → no attribution.

**Why bad:** Legal risk + trust violation + member churn.

**GOOD:** Rights-request DM → consent logged in Notion → repost with attribution + UTM.

**Why better:** Member feels respected. Audit trail exists.

---

### Antipattern 6: Ignore sentiment drop because "metric is noisy"

**BAD:** Weekly sentiment dropped 25%. Manager: "noise, ignore." Two weeks later, churn spikes. Quarter later, retention dashboard tanks.

**Why bad:** Sentiment is leading indicator. Ignoring leading indicators is how teams get blindsided.

**GOOD:** > 20% WoW decline = playbook. Investigate. Hypothesize. Communicate. Even if it turns out to be noise, the discipline pays back.

**Why better:** Early action on leading indicators saves churn.

---

### Antipattern 7: Cross-post identical content to every platform

**BAD:** Copy-paste announcement to Discord + Slack + Substack + LinkedIn + Twitter. Word-for-word.

**Why bad:** Each platform has a different voice. Cross-posting violates platform-native rule.

**GOOD:** One core message + platform-native variants (Discord short + emoji; Slack threaded + professional; Discourse long + searchable; Substack narrative).

**Why better:** Each platform performs.

---

### Antipattern 8: Mass-DM members for "engagement"

**BAD:** DM every member: "Hey! Want to chat?"

**Why bad:** Member privacy violation. Platform ToS violation. Spam classification. Bans incoming.

**GOOD:** Opt-in newsletter. Targeted, infrequent DM to specific high-context recipients.

**Why better:** Respects member time and platform rules.

## Updated mappings — replace outdated patterns

Where role.md sections name generic categories ("community platform", "community CRM", "moderation bot"), the SOTA replacement is:

- **Community platform** (Circle / Discord / Slack / Discourse / Bevy / Skool / Mighty / Heartbeat / Substack / Whop) → fit-by-audience-and-monetization, not familiarity. Default by use case in Platform selection decision matrix.
- **Community CRM** (Common Room / Notable / Threado / Orbit-legacy / HubSpot custom prop) → **Common Room** for mid-market+, **Notable / Threado** for SMB, **HubSpot custom property + Notion CRM** for free fallback.
- **Discord bots** (MEE6 / Carl-bot / Dyno / Wick / AutoMod / Statbot) → **Carl-bot + MEE6 + Dyno** as standard 2026 stack; **Wick** for high-risk anti-raid.
- **Slack bots** (Polly / Geekbot / Donut / Standuply / Workast / Birthday Bot) → **Polly + Geekbot + Donut** standard engagement stack.
- **Sentiment** (Brand24 / Talkwalker / Common Room native / Claude) → **Brand24** for cross-platform, **Common Room native** for in-community, **Claude on transcript** for free fallback.
- **Feedback portal** (Productboard / Canny / Featurebase / Frill / UserVoice / Ideawake / Aha! Ideas) → **Productboard** for enterprise, **Canny** for public roadmap, **Linear** if engineering-led.
- **Events** (Bevy / Lu.ma / Goldcast / Welcome / Hopin / Eventbrite / Zoom Events) → **Bevy** for chapter programs, **Lu.ma** for casual, **Goldcast** for B2B virtual, **Eventbrite** for ticketing only.
- **Gated community** (Memberstack / Outseta / Substack Paid / Whop / Skool / Patreon) → fit-by-use-case: SaaS = Memberstack; all-in-one = Outseta; creator-economy = Whop; course+community = Skool; newsletter = Substack.
- **Web3 community** (Snapshot / Aragon / Mirror / Lens / Farcaster / Collab.Land / Vulcan / Guild.xyz) → **Snapshot + Collab.Land + Mirror** standard; Lens / Farcaster for decentralized social.
- **Ambassador platform** (Mavrck / SocialLadder / Roster / Brandbassador / Aspire / GRIN) → **Common Room nominate + manual outreach** for SMB / mid-market; **Mavrck** for enterprise; HubSpot custom property + Notion-CRM for free fallback.
- **Swag** (Reachdesk / Sendoso / Postal / Printful / Printify / Cotton Bureau / Bonusly) → **Reachdesk** for B2B at scale, **Printful** for DTC fallback.
- **KB** (Notion / Slab / Tettra / Stack Overflow for Teams / Outline / Confluence / Mintlify / Docusaurus) → **Notion** for modern source-of-truth, **Stack Overflow for Teams** for technical Q&A, **Discourse** for community-as-SEO.

## Closing rules

Community is interaction, not broadcast. Lurkers are the majority; design for them. ROI is a leading indicator of retention. Pick the platform deliberately. Charter before launch. Welcome within minutes. Moderate as YAML. Screen ambassadors before outreach. Cultivate UGC with rights. Measure CLG like a P&L line. Defer to sibling agents (`marketing-agent`, `social-media-manager`, `customer-support-agent`, `customer-success`, `email-strategist`, `content-creator`) when the work is outside community.
