## Notes on "authored from synthesis"

Several sections in this v1 build are composition-synthesis on top of platform documentation rather than verbatim lifts from a reference agent. This is appropriate for v1 — Round 2 can backfill with concrete reference-agent files:

- **Three opening convictions in soul.md** — distillation of platform doctrine: "community is not audience — interaction is the product" (Reforge CLG framework), "lurkers > posters — design for the 90% reading silently" (Common Room + Notion community ops), "community ROI is a leading indicator of retention" (Reforge 5-dim ROI model).
- **Ambassador tier rubric** — composition synthesis from Mavrck + SocialLadder + Common Room patterns + published programs (Atlassian Community Champions, Notion Champions, Figma Friends, Hex Champions). Not lifted from a single source; the 4-tier (Member → Contributor → Ambassador → Champion) structure is canonical.
- **Charter section structure** — composition synthesis informed by Contributor Covenant 2.1 + Open Source Guide CoC pattern + community-charter examples (Stripe, Atlassian, Notion public charters).
- **Member journey stages (8-stage)** — adapted from Orbit Model (legacy reference) + Common Room funnel + Reforge community-engagement funnel.
- **ROI 5-dim model** — Reforge CLG framework verbatim; assumption-discipline language + comparison-cohort discipline + statistical-significance discipline added as composition.
- **AI-slop catch list — community edition** — adapted from marketing-agent's voltagent-content-quality-editor pattern, narrowed to community-specific phrases.
- **Antipattern catalog** — composition synthesis from common community-failure modes across Reforge / Common Room / Circle / Notion community publications.
- **First-conversation PROACTIVE.md self-init** — standard pattern from METHODOLOGY.md with community-specific routine questions (primary platform / member count / primary goal — per Round 1 prompt).

No fabricated benchmarks, fake quotes, or fake community stories. All platform features cited are real and verifiable via the URLs in `agent.yaml → sources`.
