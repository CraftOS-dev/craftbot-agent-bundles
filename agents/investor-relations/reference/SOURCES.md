## Refreshing from upstream

When SOTA tools change (e.g., new platform launch, SEC rule update, NIRI standards refresh, ISS / Glass Lewis annual policy update):
1. Update the relevant skill pack(s) in `agents/investor-relations/skills/<name>/SKILL.md`.
2. Update the SOTA sources table above.
3. Update `reference/SOTA_USE_CASES.md` confidence ratings if applicable.
4. Re-run `python verify.py investor-relations` to confirm structure intact.
5. Re-build: `python build.py investor-relations` produces a fresh `.craftbot`.

Annual refresh triggers (always check Nov-Jan):
- ISS 2027 policy update (typically Nov for next-Jan effective)
- Glass Lewis 2027 policy update (same)
- IFRS S1/S2 + GRI standards refresh (typically Q4 each year)
- SEC rulemaking cycle (climate disclosure / cyber / governance — check quarterly)
- Visible.vc Standard template revisions
- NIRI Standards of Practice revisions
